### Artifacts

- **Artifacts** are files created by a job and saved by GitLab after the job finishes.
- They can be downloaded later or passed to jobs in later stages.

---

### Artifact Workflow

```text
Build Job
    │
Creates app.jar
    │
    ▼
Artifact Stored
    │
    ▼
Test Job Downloads Artifact
```

---
### Artifact Expiration

- Artifacts can expire automatically.
- GitLab deletes the artifact after **7 days**.

---

### Download Artifacts

Artifacts can be downloaded from:

```text
Project
   │
   ▼
Build
   │
   ▼
Jobs
   │
   ▼
Download Artifacts
```

---

### Cache

- **Cache** stores files that can be reused by future pipeline runs.
- Its main purpose is to **speed up pipelines** by avoiding repeated downloads or installations.

---
### Cache Workflow

```text
First Pipeline
      │
Downloads Dependencies
      │
Stores Cache
      │
      ▼
Next Pipeline
      │
Uses Cache
      │
Faster Execution
```

---

### Key Points

- **Artifacts** store files created by a job and make them available to later jobs or for download.
- **Cache** stores reusable files to speed up future pipeline runs.
- Artifacts are commonly used for build outputs and test reports.
- Cache is commonly used for dependency folders like `node_modules` or `.m2`.
- Using artifacts and cache correctly improves pipeline efficiency and reduces execution time.
