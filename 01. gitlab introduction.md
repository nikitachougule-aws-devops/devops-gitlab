# GitLab

- It is a web-based platform used to manage Git repositories and automate the software development lifecycle.

It provides features such as:

- Git repository hosting
- Version control
- Collaboration
- Issue tracking
- Merge Requests (MR)
- CI/CD pipelines
- Code review
- Project management

---

# Why Use GitLab?

GitLab helps teams:

- Store source code.
- Collaborate with developers.
- Track issues and bugs.
- Automate build, test, and deployment.
- Manage projects in one platform.

# Common GitLab Terms

| Term | Description |
|------|-------------|
| Repository | Stores source code |
| Project | A Git repository with additional GitLab features |
| Group | Collection of related projects |
| Branch | Separate line of development |
| Merge Request (MR) | Request to merge code into another branch |
| Pipeline | Automated build, test, and deployment process |
| Runner | Executes CI/CD jobs |
| Issue | Used to track tasks and bugs |

# GitLab vs Git

| Git | GitLab |
|-----|---------|
| Version control system | Git repository hosting platform |
| Works locally | Web-based platform |
| Tracks code changes | Manages repositories and collaboration |
| Command-line tool | Web interface with DevOps features |

---

# Key Points

- GitLab is a Git repository hosting and DevOps platform.
- It provides source code management, CI/CD, code review, and project management.
- GitLab supports both cloud-hosted and self-managed deployments.
- Merge Requests are used for code reviews before merging.
- GitLab includes built-in CI/CD pipelines for automated software delivery.
