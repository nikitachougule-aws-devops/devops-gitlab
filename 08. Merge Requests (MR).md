## Merge Requests (MR)

- It is a request to merge changes from one branch into another.
- It allows team members to: Review code, Suggest improvements, Approve changes, Merge code safely
- GitLab uses **Merge Requests (MR)**, while GitHub uses **Pull Requests (PR)**.

---

### Merge Request Workflow

```text
Create Branch
      │
      ▼
Write Code
      │
      ▼
Commit Changes
      │
      ▼
Push Branch
      │
      ▼
Create Merge Request
      │
      ▼
Code Review
      │
      ▼
Approve Changes
      │
      ▼
Merge into main
```

---

### Create a Merge Request

#### Step 1: Create a Branch

```bash
git switch -c feature-login
```

---

#### Step 2: Make Changes

Edit your project files.

---

#### Step 3: Commit Changes

```bash
git add .
git commit -m "Add login feature"
```

---

#### Step 4: Push the Branch

```bash
git push origin feature-login
```

---

#### Step 5: Create the Merge Request

In GitLab:

1. Open your project.
2. Click **Merge Requests**.
3. Click **New Merge Request**.
4. Select:
   - Source branch: `feature-login`
   - Target branch: `main`
5. Add a title and description.
6. Click **Create Merge Request**.

---

### Merge Request Process

```text
Developer
     │
     ▼
Push Feature Branch
     │
     ▼
Create Merge Request
     │
     ▼
Reviewer Reviews Code
     │
     ▼
Approve or Request Changes
     │
     ▼
Merge into main
```

---

### Key Points

- A Merge Request (MR) is used to merge code from one branch into another.
- GitLab uses **Merge Requests**, while GitHub uses **Pull Requests**.
- Merge Requests enable code review, collaboration, and approval before merging.
- They help keep the `main` branch stable and improve code quality.
- A typical workflow is: **Branch → Code → Commit → Push → Merge Request → Review → Approve → Merge**.
