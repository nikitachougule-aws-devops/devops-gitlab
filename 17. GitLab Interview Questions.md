## 1. What is GitLab?

**Answer:**
GitLab is a web-based Git repository hosting platform that provides source code management, CI/CD, code review, issue tracking, and DevOps tools.

---

## 2. What is the difference between Git and GitLab?

**Answer:**

- **Git** is a version control system.
- **GitLab** is a platform that hosts Git repositories and provides CI/CD, Merge Requests, and project management.

---

## 3. What is GitLab CI/CD?

**Answer:**
GitLab CI/CD is GitLab's built-in tool that automates building, testing, and deploying applications.

---

## 4. What is a Pipeline?

**Answer:**
A pipeline is an automated workflow that runs jobs such as Build, Test, and Deploy.

---

## 5. What is a GitLab Runner?

**Answer:**
A GitLab Runner is an application that executes CI/CD pipeline jobs.

---

## 6. What is the `.gitlab-ci.yml` file?

**Answer:**
It is the configuration file that defines pipeline stages, jobs, scripts, variables, and other CI/CD settings.

---

## 7. What is a Stage?

**Answer:**
A stage is a phase in a pipeline, such as **Build**, **Test**, or **Deploy**.

---

## 8. What is a Job?

**Answer:**
A job is a single task within a stage, such as compiling code or running tests.

---

## 9. What is a Merge Request (MR)?

**Answer:**
A Merge Request is a request to merge changes from one branch into another after code review and approval.

---

## 10. What is the difference between a Stage and a Job?

**Answer:**

- **Stage** → A phase in the pipeline.
- **Job** → A task executed within a stage.

---

## 11. What happens if one job fails?

**Answer:**
The pipeline stops, and the remaining stages do not run unless configured otherwise.

---

## 12. What are GitLab Variables?

**Answer:**
Variables are key-value pairs used to store reusable values and secrets for CI/CD pipelines.

---

## 13. Why should secrets not be stored in `.gitlab-ci.yml`?

**Answer:**
Because the file is stored in the repository and may expose sensitive information. Store secrets as **GitLab CI/CD Variables** instead.

---

## 14. What is the difference between Artifacts and Cache?

**Answer:**

- **Artifacts** store files created by a job for later jobs or download.
- **Cache** stores reusable files to speed up future pipeline runs.

---

## 15. What is GitLab Container Registry?

**Answer:**
It is a built-in registry used to store and manage Docker container images.

---

## 16. What are Shared and Specific Runners?

**Answer:**

- **Shared Runner** → Used by multiple projects.
- **Specific Runner** → Dedicated to a single project.

---

## 17. What is the purpose of GitLab Groups?

**Answer:**
Groups organize related projects and simplify user and permission management.

---

## 18. What is the difference between Continuous Delivery and Continuous Deployment?

**Answer:**

- **Continuous Delivery** → Deployment requires manual approval.
- **Continuous Deployment** → Deployment happens automatically after successful tests.

---

## 19. What triggers a GitLab Pipeline?

**Answer:**

A pipeline can be triggered by:

- Git push
- Merge Request
- Tag creation
- Scheduled pipeline
- Manual execution

---

## 20. Why is my pipeline stuck in the Pending state?

**Answer:**
Usually because no GitLab Runner is available, the Runner is offline, or it is not assigned to the project.
