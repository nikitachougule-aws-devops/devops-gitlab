### Stages

- It is a phase in a GitLab CI/CD pipeline.
- Stages organize jobs into logical steps.
- Common stages are: Build, Test, Deploy
  
---

### Jobs

- It is a specific task performed within a stage.
- Each job is defined in the `.gitlab-ci.yml` file.

---

# Key Points

- A **Stage** is a phase in a pipeline, such as **Build**, **Test**, or **Deploy**.
- A **Job** is a specific task that runs within a stage.
- Jobs in the same stage can run in parallel.
- Stages run sequentially.
- If a job fails, the pipeline stops and later stages are skipped.
- Jobs and stages are defined in the `.gitlab-ci.yml` file.
