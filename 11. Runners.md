## GitLab Runners

- It is an application that executes the jobs defined in the `.gitlab-ci.yml` file.
- When a pipeline starts, the runner picks up the job, runs the commands, and sends the results back to GitLab

---

### Types of GitLab Runners

#### 1. Shared Runner

- Shared by multiple projects.
- Managed by GitLab or an administrator.
- Easy to use.
- Suitable for beginners.

---

#### 2. Specific Runner

- Assigned to a single project.
- Used for project-specific tasks.
- More secure and customizable.

---

#### 3. Group Runner

- Shared among all projects within a group.
- Useful for teams managing multiple projects.

---

### Runner Executors

An **Executor** determines where the job runs.

Common executors:

| Executor | Description |
|----------|-------------|
| Shell | Runs commands directly on the host machine |
| Docker | Runs jobs inside Docker containers |
| Kubernetes | Runs jobs in a Kubernetes cluster |
| VirtualBox | Runs jobs in a virtual machine |
| SSH | Executes jobs on a remote server |

Docker is the most commonly used executor.

---

### Install GitLab Runner (Ubuntu)

Add the repository:

```bash
curl -L https://packages.gitlab.com/install/repositories/runner/gitlab-runner/script.deb.sh | sudo bash
```

Install GitLab Runner:

```bash
sudo apt install gitlab-runner
```

---

### Register a Runner

```bash
sudo gitlab-runner register
```

You will be asked for:

- GitLab URL
- Registration Token
- Runner Name
- Executor (Shell, Docker, etc.)

---

### Check Runner Status

```bash
sudo gitlab-runner status
```

---

### Start and Stop Runner

Start:

```bash
sudo gitlab-runner start
```

Stop:

```bash
sudo gitlab-runner stop
```

Restart:

```bash
sudo gitlab-runner restart
```

---


### Key Points

- A **GitLab Runner** executes CI/CD pipeline jobs.
- GitLab creates the pipeline, while the Runner performs the work.
- Without a runner, pipeline jobs remain **Pending**.
- Runner types include **Shared**, **Specific**, and **Group**.
- Common executors are **Shell**, **Docker**, and **Kubernetes**.
- Docker is the most widely used executor for GitLab CI/CD.
