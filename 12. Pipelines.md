## GitLab Pipelines

- It is an automated workflow in GitLab CI/CD.
- It runs a series of jobs to build, test, and deploy an application whenever code is pushed or a Merge Request is created.

---

### Pipeline Workflow

```text
Developer Pushes Code
          │
          ▼
GitLab Detects Change
          │
          ▼
Creates Pipeline
          │
          ▼
Runner Executes Jobs
          │
          ▼
Build
          │
          ▼
Test
          │
          ▼
Deploy
```

---

### Pipeline Components

A pipeline consists of:

- Stages
- Jobs
- Runner

Example:

```text
Pipeline
│
├── Build Stage
│      └── Build Job
│
├── Test Stage
│      └── Test Job
│
└── Deploy Stage
       └── Deploy Job
```

---

### Pipeline Example

```yaml
stages:
  - build
  - test
  - deploy

build-job:
  stage: build
  script:
    - echo "Building application"

test-job:
  stage: test
  script:
    - echo "Running tests"

deploy-job:
  stage: deploy
  script:
    - echo "Deploying application"
```

Execution order:

```text
Build
   │
   ▼
Test
   │
   ▼
Deploy
```

If one stage fails, the next stages do not run.

---

### Pipeline Status

A pipeline can have different statuses:

| Status | Meaning |
|---------|---------|
| Pending | Waiting for a runner |
| Running | Jobs are executing |
| Passed | All jobs completed successfully |
| Failed | One or more jobs failed |
| Canceled | Pipeline was stopped manually |
| Skipped | Job or pipeline was skipped |

---

### When is a Pipeline Triggered?

A pipeline can start when:

- Code is pushed
- A Merge Request is created
- A tag is created
- A scheduled pipeline runs
- A manual pipeline is started

---

### View Pipelines

In GitLab:

1. Open your project.
2. Click **Build → Pipelines**.
3. View:
   - Pipeline status
   - Jobs
   - Logs
   - Duration

---

### Pipeline Logs

Each job generates logs.

Example:

```text
Running build...
Installing dependencies...
Build completed successfully.
```

Logs help identify and troubleshoot failures.

---

### Key Points

- A **Pipeline** is an automated workflow in GitLab CI/CD.
- Pipelines are defined in the `.gitlab-ci.yml` file.
- A pipeline contains one or more **stages** and **jobs**.
- GitLab Runner executes the pipeline jobs.
- Pipelines can be triggered by pushes, Merge Requests, schedules, or manual actions.
- If a stage fails, the remaining stages do not run.
