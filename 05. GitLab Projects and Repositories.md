## GitLab Projects and Repositories

### Project

- It is the main workspace in GitLab.
- It contains: Git repository, Source code, Branches, Merge Requests, CI/CD Pipelines, Issues, Wiki
- Every project has its own repository.

---

### Repository

-It is a storage location for your project's source code and Git history.
- It stores: Files, Folders, Commits, Branches, Tags
- Developers clone the repository to work on the project.

---

### Create a New Project

1. Log in to GitLab.
2. Click **New Project**.
3. Choose **Create Blank Project**.
4. Enter:
   - Project Name
   - Project URL
   - Visibility (Private, Internal, or Public)
5. Click **Create Project**.

---

### Clone a Repository

Copy the repository to your local machine.

```bash
git clone <repository-url>
```

Example:

```bash
git clone https://gitlab.com/username/my-project.git
```

---

### Repository Structure

```text
my-project/
│
├── .git/
├── src/
├── README.md
├── .gitignore
└── .gitlab-ci.yml
```

---

### Common Repository Operations

Check status:

```bash
git status
```

Add files:

```bash
git add .
```

Commit changes:

```bash
git commit -m "Initial commit"
```

Push changes:

```bash
git push origin main
```

Pull latest changes:

```bash
git pull origin main
```

---

### Key Points

- A **Project** is a complete workspace in GitLab.
- A **Repository** stores the project's source code and Git history.
- Every GitLab project contains a Git repository.
- Developers clone repositories to work locally.
- Projects can be **Private**, **Internal**, or **Public**.
- Common Git operations include **clone**, **add**, **commit**, **push**, and **pull**.
