## .gitlab-ci.yml

`.gitlab-ci.yml` is the configuration file used by **GitLab CI/CD**.

It defines:

- Pipeline stages
- Jobs
- Scripts
- Variables
- Rules

GitLab reads this file whenever code is pushed to the repository.

---

### Where is it Located?

The file must be placed in the root directory of the project.

```text
my-project/
│
├── .gitlab-ci.yml
├── README.md
├── app.py
└── package.json
```

---

### Basic Structure

```yaml
stages:
  - build
  - test
  - deploy

build-job:
  stage: build
  script:
    - echo "Building application"

test-job:
  stage: test
  script:
    - echo "Running tests"

deploy-job:
  stage: deploy
  script:
    - echo "Deploying application"
```

---

### Main Keywords

| Keyword | Purpose |
|----------|---------|
| `stages` | Defines pipeline stages |
| `stage` | Specifies the stage for a job |
| `script` | Commands executed by the job |
| `image` | Docker image used to run the job |
| `variables` | Defines environment variables |
| `artifacts` | Saves files between jobs |
| `rules` | Controls when jobs run |
| `only` | Run job only for specific branches or tags |
| `except` | Skip job for specific branches or tags |

---

### Example

```yaml
build:
  script:
    - echo "Hello from GitLab CI/CD"
```

When code is pushed, GitLab runs:

```bash
echo "Hello from GitLab CI/CD"
```

---

### Example with Stages

```yaml
stages:
  - build
  - test

build:
  stage: build
  script:
    - echo "Building..."

test:
  stage: test
  script:
    - echo "Testing..."
```

Execution order:

```text
Build
  │
  ▼
Test
```

---

### Key Points

- `.gitlab-ci.yml` is the configuration file for GitLab CI/CD.
- It defines **stages**, **jobs**, and **scripts**.
- GitLab reads this file automatically when code is pushed.
- A pipeline is created based on the instructions in the file.
- The file should be stored in the project's root directory.
