### Variables

- **Variables** are key-value pairs used to store reusable values.
- Instead of hardcoding values, you can store them as variables and use them in your jobs.

---

### Types of Variables

#### 1. YAML Variables

- Defined inside `.gitlab-ci.yml`.

#### 2. GitLab CI/CD Variables

- Defined in GitLab.
- These variables are stored securely and are available to pipeline jobs.

---

### Protected Variables

- A **Protected Variable** is available only to pipelines running on **protected branches** or **protected tags**.
- This prevents secrets from being exposed in feature branches.
---

### Masked Variables

- A **Masked Variable** hides its value in job logs.

---

### Secrets

- **Secrets** are sensitive variables that should not be stored directly in the `.gitlab-ci.yml` file.
- Examples: Passwords, API Keys, AWS Access Keys, SSH Private Keys, Database Credentials
- GitLab stores secrets securely as **CI/CD Variables**.
  
### Adding a Secret

Steps:

1. Open your project.
2. Go to **Settings → CI/CD**.
3. Expand **Variables**.
4. Click **Add Variable**.
5. Enter:
   - Key
   - Value
6. Save.

---

### Key Points

- Variables store reusable values for GitLab CI/CD pipelines.
- Secrets are sensitive variables such as passwords and API keys.
- Store secrets in **GitLab CI/CD Variables**, not in the repository.
- Protected variables are available only to protected branches or tags.
- Masked variables prevent sensitive values from appearing in pipeline logs.
