# GitLab Workflow

- It is the process of developing code, reviewing it, and deploying it using GitLab.
- It helps teams collaborate and automate software delivery using Git and CI/CD.

---

```text

      ▼
```

---

# Step 1: Create a Project

Create a new project in GitLab.

The project contains:

- Repository
- Branches
- Merge Requests
- Pipelines
- Issues

---

# Step 2: Clone the Repository

Download the project to your local machine.

```bash
git clone <repository-url>
cd <project-name>
```

---

# Step 3: Create a Branch

Create a branch for your feature or bug fix.

```bash
git switch -c feature-login
```

---

# Step 4: Write Code

Make the required code changes in your local project.

---

# Step 5: Commit Changes

Save your changes.

```bash
git add .
git commit -m "Add login feature"
```

---

# Step 6: Push the Branch

Upload your branch to GitLab.

```bash
git push origin feature-login
```

---

# Step 7: Create a Merge Request (MR)

In GitLab:

- Open the project.
- Click **Merge Requests**.
- Click **New Merge Request**.
- Select the source and target branches.
- Submit the Merge Request.

---

# Step 8: Code Review

Team members review the code.

They can:

- Add comments
- Request changes
- Approve the Merge Request

---

# Step 9: Merge the Code

After approval, merge the feature branch into the `main` branch.

GitLab automatically updates the repository.

---

# Step 10: CI/CD Pipeline

If a `.gitlab-ci.yml` file exists, GitLab automatically starts a pipeline.

Typical stages:

- Build
- Test
- Deploy
- Merge Requests are used for code review and approval.
- GitLab automatically runs CI/CD pipelines after code is pushed or merged.
- A standard workflow is: **Create Project → Clone → Branch → Code → Commit → Push → Merge Request → Review → Merge → Pipeline**.
