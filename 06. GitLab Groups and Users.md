## GitLab Groups and Users

### Groups
- It is a collection of related GitLab projects and users.
- Groups help organize projects and manage permissions from one place.
- Example:

```text
ABC Company
│
├── DevOps Team
│     ├── Terraform Project
│     ├── Docker Project
│     └── Kubernetes Project
│
└── Development Team
      ├── Website Project
      └── Mobile App
```

---

### Users

- It is a person who can access GitLab.
- A user can: Create projects, Clone repositories, Push code, Create Merge Requests, Review code, Run pipelines (based on permissions)

---

### Create a Group

1. Log in to GitLab.
2. Click **New Group**.
3. Enter:
   - Group Name
   - Group URL
   - Visibility
4. Click **Create Group**.

---

### Create a User

An administrator can:

1. Go to **Admin Area**.
2. Select **Users**.
3. Click **New User**.
4. Enter user details.
5. Save the user.

On GitLab.com, users usually create their own accounts.

---

### Add a User to a Group

1. Open the group.
2. Select **Manage → Members**.
3. Click **Invite Members**.
4. Select the user.
5. Assign a role.
6. Click **Invite**.

---

### Group Structure

```text
Company
   │
   ├── DevOps Team
   │      ├── Terraform
   │      ├── Docker
   │      └── Kubernetes
   │
   └── Development Team
          ├── Backend
          └── Frontend
```

---

### Key Points

- A **Group** organizes multiple GitLab projects and users.
- A **User** is a person who accesses GitLab.
- Groups simplify permission management and collaboration.
- Common roles are **Guest**, **Reporter**, **Developer**, **Maintainer**, and **Owner**.
- Use groups to organize projects by team, department, or organization.
