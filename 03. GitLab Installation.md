# GitLab Installation

## Prerequisites

Before installing GitLab, make sure you have:

- A Linux server (Ubuntu is recommended)
- At least 4 GB RAM
- 2 CPU cores
- Root or sudo access
- Internet connection

---

# GitLab Installation Options

You can use GitLab in three ways:

1. GitLab.com (Cloud)
2. Self-Managed GitLab
3. Docker

---

# Option 1: GitLab.com (Recommended for Beginners)

GitLab provides a free cloud-hosted version.

Steps:

1. Go to **https://gitlab.com**
2. Create an account.
3. Verify your email.
4. Sign in.
5. Create your first project.

No installation is required.

---

# Option 2: Install GitLab on Ubuntu

## Step 1: Update the System

```bash
sudo apt update
sudo apt upgrade -y
```

---

## Step 2: Install Required Packages

```bash
sudo apt install -y curl openssh-server ca-certificates tzdata perl
```

---

## Step 3: Add the GitLab Repository

```bash
curl https://packages.gitlab.com/install/repositories/gitlab/gitlab-ee/script.deb.sh | sudo bash
```

For the Community Edition:

```bash
curl https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.deb.sh | sudo bash
```

---

## Step 4: Install GitLab

Community Edition:

```bash
sudo EXTERNAL_URL="http://your-server-ip" apt install gitlab-ce
```

Enterprise Edition:

```bash
sudo EXTERNAL_URL="http://your-server-ip" apt install gitlab-ee
```

Replace `your-server-ip` with your server's IP address.

---

## Step 5: Access GitLab

Open your browser:

```text
http://your-server-ip
```

GitLab will display the login page.

---

## Step 6: Get the Initial Password

```bash
sudo cat /etc/gitlab/initial_root_password
```

Example:

```text
Password: xxxxxxxxxxxxxx
```

Login using:

- Username: `root`
- Password: Initial password

---

# Option 3: Install GitLab Using Docker

Pull the GitLab image:

```bash
docker pull gitlab/gitlab-ce:latest
```

Run the container:

```bash
docker run --detach \
  --hostname gitlab.local \
  --publish 80:80 \
  --publish 443:443 \
  --publish 22:22 \
  --name gitlab \
  gitlab/gitlab-ce:latest
```

Access GitLab:

```text
http://localhost
```

---

# Verify Installation

Check the GitLab version:

```bash
gitlab-rake gitlab:env:info
```

Check GitLab status:

```bash
sudo gitlab-ctl status
```

---

# GitLab Services

Manage GitLab services:

Start:

```bash
sudo gitlab-ctl start
```

Stop:

```bash
sudo gitlab-ctl stop
```

Restart:

```bash
sudo gitlab-ctl restart
```

Status:

```bash
sudo gitlab-ctl status
```
