# GitLab Architecture

- It describes how GitLab components work together to manage source code, automate CI/CD, and deploy applications.

The main components are:

- GitLab Server
- Git Repository
- GitLab Runner
- CI/CD Pipeline
- Developer

---

# Components of GitLab

## 1. Developer

The developer:

- Clones the repository.
- Creates a new branch.
- Writes code.
- Commits changes.
- Pushes code to GitLab.
- Creates a Merge Request.

---

## 2. GitLab Server

The GitLab Server is the central platform.

It manages:

- Projects
- Repositories
- Users
- Groups
- Merge Requests
- Pipelines
- Issues

---

## 3. Git Repository

A Git repository stores:

- Source code
- Branches
- Commit history
- Tags

Every GitLab project contains at least one Git repository.

---

## 4. Merge Request (MR)

A Merge Request is used to:

- Review code.
- Discuss changes.
- Approve updates.
- Merge code into another branch.

---

## 5. CI/CD Pipeline

A pipeline automates:

- Build
- Test
- Deploy

It starts automatically after a push or Merge Request.

---

## 6. GitLab Runner

A GitLab Runner executes pipeline jobs.

It performs tasks such as:

- Compiling code
- Running tests
- Building Docker images
- Deploying applications

---

# GitLab Request Flow

```text
Developer
     │
     ▼
Git Push
     │
     ▼
GitLab Server
     │
     ▼
Pipeline Created
     │
     ▼
Runner Executes Jobs
     │
     ▼
Application Deployed
```

- CI/C pipelines automate build, test, and deployment.
- GitLab Runners execute pipeline jobs.
- GitLab supports cloud-hosted, self-managed, and dedicated deployments.
